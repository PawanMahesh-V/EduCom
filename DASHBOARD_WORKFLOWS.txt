================================================================================
1. ADMIN DASHBOARD WORKFLOW
================================================================================

LOGIN:
- Navigate to /login
- Enter credentials (email: admin@szabist.pk, password)
- System validates and redirects to /admin

SECTIONS AVAILABLE:
├── Overview
├── User Management
├── Course Management
└── Direct Messages

---

A. OVERVIEW SECTION:
-------------------
Purpose: View system statistics and monitor platform activity

Workflow:
1. Dashboard loads with stat cards showing:
   - Total Users (with breakdown by role)
   - Total Courses
   - Active Communities
   - Total Notifications

Features:
- Real-time stats
- Auto-refresh on updates
- Visual representation with icons

---

B. USER MANAGEMENT SECTION:
---------------------------
Has 2 tabs: Add Users | Approve Requests

TAB 1: ADD USERS
----------------
Purpose: Create new users or manage existing users

Workflow - Add New User:
1. Click "Add New User" tab
2. Click "+" button
3. Fill form:
   - Registration ID
   - Full Name
   - Email (must end with @szabist.pk)
   - Password
   - Role (Admin/Teacher/Student/HOD/PM)
   - Department (CS/BBA/IT)
   - Semester (for Students: 1-8)
   - Program Year (for PM: 1-4)
4. Click "Create User"
5. System validates email domain
6. User created and welcome notification sent
7. Modal closes, user list refreshes

Workflow - Edit Existing User:
1. View user list (with filters)
2. Apply filters:
   - Search by name/email/ID
   - Filter by Role
   - Filter by Department
3. Click "Edit" button on user row
4. Modify user details
5. Click "Update"
6. User updated and list refreshes

Workflow - Delete User:
1. Click "Delete" button on user row
2. Confirm deletion
3. User removed from system

TAB 2: APPROVE REQUESTS
-----------------------
Purpose: Review and approve/reject user registration requests

Workflow:
1. Click "Approve Requests" tab
2. View pending registration requests table showing:
   - Registration ID
   - Name
   - Email
   - Role
   - Department
   - Semester/Year
   - Request Date
3. Review request details
4. Take action:

   Option A - Approve:
   - Click green checkmark button (Approve)
   - System creates user account
   - Sends approval email to user
   - Sends welcome notification
   - Request marked as approved
   - List refreshes

   Option B - Reject:
   - Click red X button (Reject)
   - Request marked as rejected
   - List refreshes

Features:
- Desktop: Table view
- Mobile: Card view
- Real-time updates

---

C. COURSE MANAGEMENT SECTION:
-----------------------------
Has 3 tabs: Courses | Communities | Approve Courses

TAB 1: COURSES
--------------
Purpose: Manage all courses in the system

Workflow - Add New Course:
1. Click "Courses" tab
2. Click "+" button
3. Fill form:
   - Course Code (e.g., CS101)
   - Course Name
   - Department (CS/BBA/IT)
   - Semester (1-8)
   - Assign Teacher
4. Click "Create Course"
5. System:
   - Creates course
   - Generates random 8-character join code
   - Creates community automatically
   - Links community to course
6. Modal closes, course list refreshes

Workflow - Edit Course:
1. Apply filters (Department/Semester) or search
2. Click "Edit" button on course
3. Modify course details
4. Click "Update"
5. Course updated, list refreshes

Workflow - Delete Course:
1. Click "Delete" button on course
2. Confirm deletion
3. Course and associated community removed

Features:
- Desktop: Table view with actions
- Mobile: Card view
- Search by code/name/department
- Filter by department and semester

TAB 2: COMMUNITIES
------------------
Purpose: Manage course communities and send announcements

Workflow - Edit Community:
1. Click "Communities" tab
2. View communities list
3. Apply filters (Status/Search)
4. Click "Edit" button
5. Update community name or status
6. Click "Update"

Workflow - Send Message to Community:
1. Click "Send Message" button on community
2. Fill message form:
   - Subject
   - Message content
3. Click "Send Message"
4. Message broadcast to all community members
5. Notifications sent to all enrolled students

Features:
- Filter by status (Active/Inactive)
- Search by name/course code
- Bulk messaging capability

TAB 3: APPROVE COURSES
---------------------
Purpose: Review and approve course requests from teachers

Workflow:
1. Click "Approve Courses" tab
2. View pending course requests showing:
   - Course Code
   - Course Name
   - Department
   - Semester
   - Teacher (who will teach)
   - Requested By (who submitted)
3. Review request details
4. Take action:

   Option A - Approve:
   - Click green checkmark button
   - System:
     * Creates course
     * Generates join code
     * Creates community
     * Links everything together
     * Sends notification to teacher with join code
     * Marks request as approved
   - List refreshes

   Option B - Reject:
   - Click red X button
   - Request rejected
   - List refreshes

Features:
- Desktop: Table view
- Mobile: Card view
- Shows requester information

---

D. DIRECT MESSAGES SECTION:
--------------------------
Purpose: Private messaging with users

Workflow - Start New Conversation:
1. Click "Direct Messages" section
2. Click "New Message" button
3. Search for user by name/email
4. Select user from list
5. Type message
6. Click send or press Enter
7. Message delivered instantly

Workflow - Reply to Conversation:
1. View conversations list (shows unread count)
2. Click on conversation
3. Messages load
4. Type reply
5. See typing indicator when other user types
6. Send message
7. Conversation updates in real-time

Features:
- Unread message badges
- Typing indicators
- Real-time message delivery
- Search conversations
- Message timestamps

A. MY COURSES SECTION:
---------------------

1. Dashboard loads showing course cards
2. Each card displays:
   - Course Code
   - Course Name
   - Join Code (with copy button)

1. Click "+" button (top right)
2. Fill course request form:
   - Course Code
   - Course Name
   - Department (CS/BBA/IT)
   - Semester (1-8)

4. Request sent to admin for approval
5. Success message shown
6. Modal closes

Workflow - After Course Approved:
1. Receive notification: "Course Request Approved"
2. Notification includes course code and join code
3. Course appears in "My Courses"
4. Community automatically created
5. Can start managing students

Features:
- Copy join code to clipboard
- Visual course cards
- Auto-refresh when courses updated
- Request tracking

B. JOIN COURSE SECTION:
-----------------------
Purpose: Enroll in new courses using join codes

Workflow:
1. Click "Join Course" section
2. View input field for join code
3. Enter 8-character join code from teacher
4. Click "Join Course" button
5. System validates:
   - Code exists
   - Not already enrolled
   - Department matches (CS course → CS student)
6. If valid:
   - Student enrolled in course
   - Added to course community
   - Success message shown
   - Course appears in "My Courses"
7. If invalid:
   - Error message shown
   - Prompt to check code

Validation Rules:
- Must be valid 8-character code
- Cannot join same course twice
- Department must match course department
- Code must belong to active course

================================================================================
5. KEY WORKFLOWS BY USER JOURNEY
================================================================================

NEW STUDENT REGISTRATION:
1. Student visits /register
2. Fills registration form
3. Submits request
4. Admin receives request in "Approve Requests"
5. Admin reviews and approves
6. Student receives approval email
7. Student logs in
8. Gets join code from teacher
9. Joins course via "Join Course"
10. Accesses course materials and community

TEACHER COURSE CREATION:
1. Teacher logs in
2. Goes to "My Courses"
3. Clicks "+" to request course
4. Fills course details (auto-assigned to self)
5. Submits request
6. Admin sees in "Approve Courses"
7. Admin approves
8. System creates course + join code + community
9. Teacher receives notification with join code
10. Teacher shares join code with students
11. Students join using code
12. Teacher can communicate via community chat

ADMIN COURSE MANAGEMENT:
1. Admin logs in
2. Goes to "Course Management"
3. Option A - Manual creation:
   - Clicks "+" in Courses tab
   - Fills form and assigns teacher
   - Course created with join code
4. Option B - Approval:
   - Goes to "Approve Courses" tab
   - Reviews teacher request
   - Approves request
   - Course auto-created with join code
5. Course visible to assigned teacher
6. Community auto-created
7. Join code generated

MESSAGING FLOW:
1. Sender opens messages section
2. Selects recipient/community
3. Types message
4. Socket.IO sends message to server
5. Server broadcasts to recipients
6. Recipients receive instantly
7. Notification sent if offline
8. Unread count updated
